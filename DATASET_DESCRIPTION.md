Generated by `generate_dataset_clip.py` from the MOCHEG corpus.  
Paper basis: Papadopoulos et al., _Synthetic Misinformers_, MAD '23.

## Misinformation Types

The dataset contains exactly four types of rows, each corresponding to a generation strategy from the Synthetic Misinformers paper.

### `Truthful`

The original, unmodified (image, claim, evidence) triple from MOCHEG.  
Serves as the positive-class baseline for evaluation.

- Image: the claim's own proof image (`evidence_id` from `img_evidence_qrels.csv`)
- Claim: original MOCHEG claim text
- `generated_misleading_claim` = `original_claim` (identical, no manipulation)

### `Misleading-OOC` — Out-of-Context

The claim text is kept unchanged, but the proof image is **swapped** with an image from a semantically similar but factually different claim.

**Strategy: CSt-alt (CLIP image-image + text-text alternating Top-K)**

1. Encode all claim texts and proof images using CLIP ViT-L/14
2. For each query claim `i`, build an alternating Top-K neighbour list by interleaving image-image and text-text cosine similarity rankings
3. Select the highest-ranked donor claim `j` (j ≠ i) whose image differs from the query's image
4. Replace `image_path` with the donor's proof image

This creates **hard negatives** — the swapped image is visually and thematically plausible for the claim, making the pairing difficult to identify as misleading. This is precisely why OOC accuracy (38.4%) is the lowest of all techniques.

### `Misleading-NEI` — Named Entity Inconsistency

The proof image is kept unchanged, but one **named entity** in the claim text is replaced with a different entity of the same type from a semantically similar donor claim.

**Strategy: CLIP-NESt-alt (CLIP-guided entity swap)**

1. Build the same alternating Top-K neighbour list as OOC
2. Walk the ranked donor list to find the first donor `j` that:
   - Shares at least one entity **type** with query claim `i`
   - Has a different entity **value** for that type
3. Priority order for entity swapping: `PERSON > LOC > ORG > DATE > EVENT`
4. Replace the query entity in `original_claim` to produce `generated_misleading_claim`

If no swappable entity is found (3.4% of cases), the claim is tagged `FALLBACK` and a context disclaimer is appended.

### `Misleading-Hybrid` — OOC + NEI combined

Both manipulations applied simultaneously:

- Image swapped (same CSt-alt OOC strategy)
- Named entity replaced (same CLIP-NESt-alt NEI strategy)

This requires a fact-checking model to detect inconsistencies in **both** the visual and textual channels.

## Column Reference

| #   | Column                       | Type   | Non-empty for | Description                                                                                                                                                                                                     |
| --- | ---------------------------- | ------ | ------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | `claim_id`                   | string | All           | MOCHEG claim identifier (e.g. `"6"`, `"17"`)                                                                                                                                                                    |
| 2   | `original_claim`             | string | All           | The original unmodified claim text from MOCHEG                                                                                                                                                                  |
| 3   | `image_path`                 | string | All           | Filename of the proof image shown to the model. For `Truthful`/`NEI` rows this is the claim's own proof image. For `OOC`/`Hybrid` rows this is the **donor** image. Join with `images_dir` to get the full path |
| 4   | `grouped_text_evidence`      | string | All           | All text evidence paragraphs for this claim joined by `\|\|\|`. Sourced from MOCHEG `Corpus2.csv`. May be long (hundreds of words)                                                                              |
| 5   | `misinformation_label`       | string | All           | One of: `Truthful` / `Misleading-OOC` / `Misleading-NEI` / `Misleading-Hybrid`                                                                                                                                  |
| 6   | `generated_misleading_claim` | string | All           | The claim text that was **shown to the model** during evaluation. For `Truthful` and `OOC` rows this equals `original_claim`. For `NEI` and `Hybrid` rows this contains the entity-swapped version              |
| 7   | `technique`                  | string | All           | Internal technique identifier: `Truthful` / `OOC-CSt-alt-CLIP` / `NEI-CLIP-NESt-alt` / `Hybrid-CLIP-NESt-alt+CSt-alt`                                                                                           |
| 8   | `original_truthfulness`      | string | All           | The MOCHEG ground-truth label: `Supported`, `Refuted`, or `NEI` (Not Enough Information). Use this to derive `new_label` for evaluation                                                                         |
| 9   | `ooc_donor_claim_id`         | string | OOC, Hybrid   | The `claim_id` of the claim whose proof image was used as the swapped image                                                                                                                                     |
| 10  | `ooc_original_image`         | string | OOC, Hybrid   | The proof image filename that originally belonged to this claim (before the swap)                                                                                                                               |
| 11  | `ooc_swapped_image`          | string | OOC, Hybrid   | The donor proof image filename that was swapped in. This equals `image_path` for OOC/Hybrid rows                                                                                                                |
| 12  | `nei_swapped_entity_type`    | string | NEI, Hybrid   | Entity type that was swapped: `PERSON`, `LOC`, `ORG`, `DATE`, `EVENT`, or `FALLBACK`                                                                                                                            |
| 13  | `nei_swapped_from`           | string | NEI, Hybrid   | The original entity value in the claim (e.g. `"Joe Biden"`)                                                                                                                                                     |
| 14  | `nei_swapped_to`             | string | NEI, Hybrid   | The replacement entity value from the donor (e.g. `"Donald Trump"`)                                                                                                                                             |
| 15  | `nei_donor_claim_id`         | string | NEI, Hybrid   | The `claim_id` whose named entity was used as the replacement                                                                                                                                                   |
| 16  | `clip_similarity_score`      | string | _(reserved)_  | Currently empty. Reserved for CLIP cosine similarity score between query and donor                                                                                                                              |

## NER Engine

Named entity recognition uses **spaCy `en_core_web_trf`** (transformer-based, CoNLL F1 = 0.90) when available.

spaCy entity labels are mapped to canonical types as follows:

| spaCy Label                    | Mapped Type |
| ------------------------------ | ----------- |
| `PERSON`                       | `PERSON`    |
| `GPE` (geo-political entity)   | `LOC`       |
| `LOC`                          | `LOC`       |
| `FAC` (facility)               | `LOC`       |
| `ORG`                          | `ORG`       |
| `NORP` (nationalities, groups) | `ORG`       |
| `EVENT`                        | `EVENT`     |
| `DATE`                         | `DATE`      |
| `TIME`                         | `DATE`      |
| `ORDINAL`                      | `DATE`      |
| `CARDINAL`                     | `DATE`      |

If spaCy is unavailable, the script falls back to hand-crafted regex patterns covering common US locations, dates, and person name patterns.
